6/// Given the length in bytes of either a complete input or a subtree input, return the number of
7/// bytes that belong to its left child subtree. The rest belong to its right child subtree.
8///
9/// Concretely, this function returns the largest power-of-two number of bytes that's strictly less
10/// than `input_len`. This leads to a tree where all left subtrees are "complete" and at least as
11/// large as their sibling right subtrees, as specified in section 2.1 of [the BLAKE3
12/// paper](https://github.com/BLAKE3-team/BLAKE3-specs/blob/master/blake3.pdf). For example, if an
13/// input is exactly two chunks, its left and right subtrees both get one chunk. But if an input is
14/// two chunks plus one more byte, then its left subtree gets two chunks, and its right subtree
15/// only gets one byte.
16///
17/// This function isn't meaningful for one chunk of input, because chunks don't have children. It
18/// currently panics in debug mode if `input_len <= CHUNK_LEN`.
19#[inline(always)]
20pub(super) const fn left_subtree_len(input_len: u64) -> u64 {
21    debug_assert!(input_len > CHUNK_LEN as u64);
22    // Note that .next_power_of_two() is greater than *or equal*.
23    input_len.div_ceil(2).next_power_of_two()
24}
